# HW-10 Dynamic Programming

## วิธีตีโจทย์ DP (Template)

```
1. นิยาม:     dp[i] คืออะไร?
2. Base Case: dp[?] = ?
3. สูตร:      dp[i] = f(dp[i-?], ...)
4. Loop:      วนทิศทางไหน?
5. คำตอบ:     return dp[?]
```

---

## สรุปทุกไฟล์

---

### 1. Bottom-Up.c++
**โจทย์:** หาค่า G(n) โดย G(n) = G(n-1) + G(n-2) - G(n-3)

| | |
|--|--|
| **วิธี** | Bottom-Up DP |
| **dp[i]** | ค่าของ G(i) |
| **Base Case** | dp[0]=0, dp[1]=1, dp[2]=1 |
| **สูตร** | `dp[i] = dp[i-1] + dp[i-2] - dp[i-3]` |
| **คำตอบ** | `dp[n]` |

---

### 2. Top-Down.c++
**โจทย์:** หาค่า F(n, k) โดย F(n,k) = F(n-1,k) + F(n-1,k-2)

| | |
|--|--|
| **วิธี** | Top-Down DP (Memoization + Recursion) |
| **dp[n][k]** | ค่าของ F(n,k) |
| **Base Case** | k==0, k==1, k==n → return 1 |
| **สูตร** | `F(n,k) = F(n-1,k) + F(n-1,k-2)` |
| **คำตอบ** | `F(n,k)` |

```
ขั้นตอน Top-Down:
1. ตรวจ base case
2. ตรวจ memo → ถ้ามีแล้วคืนค่าเลย
3. คำนวณจากปัญหาย่อย
4. เก็บ memo แล้วคืนค่า
```

---

### 3. Grid.c++
**โจทย์:** นับจำนวนเส้นทางจาก (1,1) ถึง (n,m) เดินได้เฉพาะขวาและลง

| | |
|--|--|
| **วิธี** | Bottom-Up 2D DP |
| **dp[i][j]** | จำนวนเส้นทางจาก (1,1) ถึง (i,j) |
| **Base Case** | แถวแรก/คอลัมน์แรก = 1 |
| **สูตร** | `dp[i][j] = dp[i-1][j] + dp[i][j-1]` |
| **คำตอบ** | `dp[n][m]` |

---

### 4. KnapsackMaxitems.c++
**โจทย์:** มีของ n ชิ้น แต่ละชิ้นมีราคา price[i] มีเงิน k บาท ต้องการซื้อของได้มากชิ้นที่สุด

| | |
|--|--|
| **วิธี** | 0/1 Knapsack Bottom-Up |
| **dp[j]** | จำนวนชิ้นสูงสุดที่ซื้อได้ด้วยเงิน j บาท |
| **Base Case** | dp[0]=0, dp[อื่นๆ]=-1 (ยังซื้อไม่ได้) |
| **สูตร** | `dp[j] = max(dp[j], dp[j-price[i]] + 1)` |
| **Loop** | วนจาก k ลงมา (วนถอยหลัง) เพื่อไม่ให้ใช้ของซ้ำ |
| **คำตอบ** | `dp[k]` |

---

### 5. Min-Top-Down.c++
**โจทย์:** หา minimum cost P(n,m) โดยเดินได้ 3 ทิศ พร้อม cost แต่ละทิศ

| | |
|--|--|
| **วิธี** | Top-Down DP (Memoization) |
| **dp[i][j]** | minimum cost ในการไปถึง (i,j) |
| **Base Case** | i==0 → 3*j, j==0 → 2*i |
| **สูตร** | `P(i,j) = min(P(i-1,j)+2, P(i,j-1)+3, P(i-1,j-1)+C(i,j))` |
| **C(i,j)** | cost แนวทแยง = 2i+j |
| **คำตอบ** | `P(n,m)` |

---

### 6. Minimum_Path_Matrix.c++
**โจทย์:** หา minimum cost จากจุด (x1,y1) ถึง (x2,y2) ในเมทริกซ์ เดินได้เฉพาะขวาและลง

| | |
|--|--|
| **วิธี** | Top-Down DP (Memoization) เดินย้อนกลับ |
| **dp[i][j]** | minimum cost จาก (x1,y1) ถึง (i,j) |
| **Base Case** | i==x1 && j==y1 → คืน cost[x1][y1] |
| **สูตร** | `S(i,j) = min(S(i-1,j), S(i,j-1)) + cost[i][j]` |
| **คำตอบ** | `S(x2,y2)` |

> **สังเกต:** Recursion เดิน **ย้อนกลับ** (จากปลายหา Base) ต่างจาก Bottom-Up

---

### 7. Rod-Cutting-Problem.c++
**โจทย์:** ตัดท่อนไม้ยาว n ให้ได้กำไรสูงสุด แต่ละความยาวมีราคากำหนดให้

| | |
|--|--|
| **วิธี** | Bottom-Up DP |
| **dp[i]** | กำไรสูงสุดจากท่อนยาว i |
| **Base Case** | dp[0]=0 |
| **สูตร** | `dp[i] = max(price[j] + dp[i-j])` สำหรับ j=1 ถึง i |
| **คำตอบ** | `dp[n]` |

---

### 8. Coin-Change-Problem.c++
**โจทย์:** มีเหรียญ k ชนิด แต่ละชนิดใช้ได้ไม่จำกัด นับจำนวนวิธีที่รวมได้ยอด n

| | |
|--|--|
| **วิธี** | Bottom-Up DP (Combination) |
| **dp[j]** | จำนวนวิธีที่รวมเหรียญได้ยอด j |
| **Base Case** | dp[0]=1 (ไม่หยิบเหรียญเลย) |
| **สูตร** | `dp[j] += dp[j - coin[i]]` |
| **Loop** | เหรียญนอก, ยอดใน → นับแบบ combination (ไม่ซ้ำ) |
| **คำตอบ** | `dp[n]` |

---

## เปรียบเทียบ Top-Down vs Bottom-Up

| | Top-Down | Bottom-Up |
|--|----------|-----------|
| **วิธีเขียน** | Recursion + Memo | Loop |
| **เริ่มจาก** | ปัญหาใหญ่ → เล็ก | เล็ก → ใหญ่ |
| **ข้อดี** | เขียนง่าย, คำนวณเฉพาะที่ใช้ | เร็วกว่า, ไม่มี stack overflow |
| **ไฟล์ในงานนี้** | Top-Down, Min-Top-Down, Minimum_Path_Matrix | Bottom-Up, Grid, KnapsackMaxitems, Rod-Cutting, Coin-Change |
